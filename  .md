# AgentForge — Codex Fix Spec

## Project Overview

AgentForge is a React dashboard for orchestrating AI agent swarms. It has 3 TypeScript projects:

- **Browser** (`src/`) — Vite + React 19 + Tailwind 4
- **Server** (`server/`) — Express API + Drizzle ORM (Postgres)
- **Worker** (`worker/`) — Background task executor (polling loop)

Architecture: Browser → Supabase (Postgres + auth + real-time) → Worker (Railway)

**Branch**: `feat/supabase-backend`

**TS config rules**: `verbatimModuleSyntax: true`, `erasableSyntaxOnly: true` — use `import type` for type-only imports, no enums in new code.

## Current Errors to Fix

### 1. Frontend: `use-swarm.ts` start() inserts invalid agent_tasks

**File**: `src/hooks/use-swarm.ts` lines 204–229

The `start()` function inserts hardcoded string `agent_id` values (`'scout-alpha'`, `'scout-beta'`, etc.) into the `agent_tasks` table. But the DB schema (`server/db/schema.ts` line 156) defines `agent_id` as `uuid` with a foreign key to `agents.id`. This causes a FK constraint violation → **500 error**.

Additionally, the insert includes columns that don't exist in the `agent_tasks` schema:
- `agent_name` — not in schema
- `role` — not in schema
- `domain` — not in schema
- `utilization` — not in schema
- `ticks_remaining` — not in schema
- `dependencies` — not in schema

The actual `agent_tasks` columns are: `id`, `swarm_run_id`, `agent_id`, `status`, `phase`, `input`, `output`, `tokens_used`, `leased_at`, `lease_expires_at`, `lease_token`, `attempts`, `max_attempts`, `last_error`, `created_at`, `completed_at`.

**Fix**: The `start()` function needs to:
1. First query existing agents from the `agents` table for the current user
2. If no agents exist, create default ones via insert into `agents` table
3. Then insert `agent_tasks` using the real UUID `agent_id` values and only the columns that exist in the schema
4. Put the prompt/context into the `input` JSONB column instead of nonexistent columns

### 2. Frontend: `AgentTaskRow` interface doesn't match schema

**File**: `src/hooks/use-swarm.ts` lines 37–48

The `AgentTaskRow` interface includes fields not in the DB (`agent_name`, `role`, `domain`, `utilization`, `ticks_remaining`, `dependencies`). The task-to-agent mapping should come from the join `agent_tasks` + `agents` (which is already done at line 87–88 with `select('*, agents(name, role)')`), but the row type needs to match reality.

### 3. TypeScript compilation errors (browser)

```
src/components/memory-inspector/MemoryInspector.tsx(300,49): error TS6133: 'backendLoading' is declared but its value is never read.
src/components/vector-galaxy/VectorGalaxy.tsx(1,33): error TS6133: 'ThreeEvent' is declared but its value is never read.
```

**Fix**: Remove the unused `backendLoading` destructuring and the unused `ThreeEvent` import.

### 4. Server: `result.rowCount` may be undefined

**File**: `server/src/dashboard/queries.ts` lines 67, 121, 173

Drizzle's `.delete()` returns a result where `rowCount` can be `number | undefined`. The comparisons `result.rowCount > 0` will fail when `rowCount` is undefined.

**Fix**: Use `(result.rowCount ?? 0) > 0` or change return type handling.

### 5. Worker: `claim_next_task` RPC may not exist

**File**: `worker/src/executor.ts` line 677

The worker calls `supabase.rpc('claim_next_task')` which is a Postgres function that must be created via migration. If it doesn't exist, the worker logs a FATAL error and won't execute any tasks.

The RPC should atomically:
1. Find the oldest `pending` task
2. Set status to `active`, set `leased_at`, `lease_expires_at`, `lease_token`, increment `attempts`
3. Return `{ id, lease_token }`

**Fix**: Ensure the SQL migration creates this function. Check `server/db/migrations/` for existing migration or create one.

## Schema Reference (agent_tasks)

```sql
CREATE TABLE agent_tasks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  swarm_run_id UUID NOT NULL REFERENCES swarm_runs(id) ON DELETE CASCADE,
  agent_id UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
  status task_status NOT NULL DEFAULT 'pending',
  phase swarm_phase NOT NULL,
  input JSONB,           -- { prompt, context?, tools?, dependencies? }
  output JSONB,          -- { response?, toolResults?, memoriesStored? }
  tokens_used INTEGER,
  leased_at TIMESTAMPTZ,
  lease_expires_at TIMESTAMPTZ,
  lease_token UUID,
  attempts INTEGER NOT NULL DEFAULT 0,
  max_attempts INTEGER NOT NULL DEFAULT 3,
  last_error TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  completed_at TIMESTAMPTZ
);
```

## Schema Reference (claim_next_task RPC)

This function needs to exist in Postgres:

```sql
CREATE OR REPLACE FUNCTION claim_next_task()
RETURNS TABLE(id UUID, lease_token UUID)
LANGUAGE plpgsql AS $$
DECLARE
  v_task_id UUID;
  v_lease_token UUID := gen_random_uuid();
BEGIN
  SELECT at.id INTO v_task_id
  FROM agent_tasks at
  WHERE at.status = 'pending'
    AND at.attempts < at.max_attempts
  ORDER BY at.created_at ASC
  LIMIT 1
  FOR UPDATE SKIP LOCKED;

  IF v_task_id IS NULL THEN
    RETURN;
  END IF;

  UPDATE agent_tasks
  SET status = 'active',
      leased_at = now(),
      lease_expires_at = now() + interval '5 minutes',
      lease_token = v_lease_token,
      attempts = attempts + 1
  WHERE agent_tasks.id = v_task_id;

  RETURN QUERY SELECT v_task_id, v_lease_token;
END;
$$;
```
